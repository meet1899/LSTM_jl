## 3. Fix the ML pipeline before making the app

This is the most important step.

### A. Remove data leakage

Right now the scaler should not be fit on the full dataset.
Fit it on **training data only**, then transform validation and test data. For time series, future values must not affect training preprocessing. scikit-learnâ€™s `TimeSeriesSplit` exists specifically because ordinary random splitting is inappropriate for time-ordered data. ([Scikit-learn][2])

### B. Add proper train / validation / test logic

Use something like:

* 70% train
* 15% validation
* 15% test

Or better:

* **walk-forward validation**
* `TimeSeriesSplit`

### C. Add real evaluation metrics

At minimum:

* MAE
* RMSE
* MAPE
* direction accuracy

### D. Add baseline models

Before trusting LSTM, compare against:

* previous-day close
* moving average baseline
* linear regression / XGBoost baseline

If LSTM does not beat those, the app should say so.

## 5. Improve the model training process

Keep your LSTM, but make the training more professional.

### Add:

* `EarlyStopping`
* `ModelCheckpoint`
* fixed random seeds
* config file for hyperparameters

Keras provides built-in callbacks like `EarlyStopping`, which can stop training when validation performance stops improving, and `ModelCheckpoint`, which saves model snapshots. ([TensorFlow][3])

### Recommended first production version

* LSTM model
* naive baseline
* one non-deep-learning baseline

That gives you comparison and credibility.
