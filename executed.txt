Changes So Far

We turned the project from a mostly notebook-style setup into a working `src/` training pipeline for the early parts of the plan.

In [preprocessing.py](/C:/Users/meetp/files/PROJECTS/LSTM_jl/src/preprocessing.py), I implemented the Step 3A foundation:
- `chronological_split(...)` splits the dataframe into `train`, `val`, and `test` in time order, with no shuffling.
- `scale_splits_train_only(...)` fits `StandardScaler` only on training data, then transforms validation and test using the train-fitted scalers.
- `assert_no_scaler_leakage(...)` checks that scaler statistics actually match training-only data.
This fixed the original leakage problem from the notebook, where scaling was done on the full dataset before splitting.

In [sequence.py](/C:/Users/meetp/files/PROJECTS/LSTM_jl/src/sequence.py), I added reusable sequence creation:
- `create_sequences(...)` builds `(X, y)` windows from a split.
- `create_sequences_with_past_context(...)` lets validation and test use only prior history for their lookback windows.
This matters because the first validation/test samples still need prior context, but they must not see future rows.

In [train.py](/C:/Users/meetp/files/PROJECTS/LSTM_jl/src/train.py), I first added `PreparedTrainingData` and `prepare_training_data(...)` so the whole no-leakage split/scale/sequence flow can be called from one place. That completed Step 3A in modular form.

Then I implemented Step 3B in the same file:
- `TrainingConfig` from [config.py](/C:/Users/meetp/files/PROJECTS/LSTM_jl/src/config.py) is used to control lookback, split ratios, epochs, batch size, seed, and early stopping patience.
- `set_random_seed(...)` makes runs more reproducible.
- `_require_tensorflow()` gives a clean error if TensorFlow is missing.
- `build_lstm_model(...)` defines the baseline LSTM.
- `train_model(...)` trains on `x_train, y_train` and validates on `x_val, y_val`.
- `train_and_evaluate(...)` evaluates only after training is done.
- `run_training_pipeline(...)` loads a CSV and runs the whole pipeline.
This changed the workflow from “just prepare arrays” to “actually train with validation and reserve test for final evaluation.”

In [evaluate.py](/C:/Users/meetp/files/PROJECTS/LSTM_jl/src/evaluate.py), I implemented Step 3C:
- `inverse_transform_targets(...)` converts scaled values back to real prices.
- `direction_accuracy(...)` measures whether the model gets the up/down movement right.
- `regression_metrics(...)` now returns `mae`, `rmse`, `mape`, and `direction_accuracy`.
- `SplitEvaluation` stores metrics plus original-scale `actuals` and `predictions`.
- `evaluate_split(...)` evaluates model outputs.
- `format_metrics(...)` prints a readable summary.
This made evaluation more realistic and useful, not just training loss.
